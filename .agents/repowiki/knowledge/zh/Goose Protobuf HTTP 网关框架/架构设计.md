- **根包作为跨子模块契约层**：`RouteInfo`/`ExtractRouteInfo`/`InjectRouteInfo`、`BreakOnError`/`ContinueOnError`、`ValidateRequest` 接口以及 `url.Values` 中转格式由根包统一导出，server、client、middleware、outgoing 及 protoc_gen_goose 生成的 Handler 均依赖同一套上下文键与错误聚合语义。
- **编译期—运行期解耦**：`protoc_gen_goose` 插件在编译期把 `.proto` + googleapis HTTP 规则映射为 Go Handler/Client/OpenAPI；运行期由 `server` 负责解码/编码与中间件链调用，`client` 与 `outgoing` 复用相同编解码路径完成反向请求。
- **Makefile 驱动的一体化流水线**：`make install` 安装本地构建的 `protoc-gen-goose`，`make example` 通过 `--goose_out` 对 `example/*/*.proto` 执行代码生成与 OpenAPI 导出，`make all` 串联两者，确保示例始终与当前源码版本同步。
- **共享第三方协议定义**：`third_party/google` 目录提供 `google/api/http.proto` 等外部 proto，使所有子模块在同一份 HTTP 规则语义下工作。
