- **核心模式**：采用 Builder 模式与接口隔离原则（ISP），通过 `MethodSetter` -> `URLSetter` -> `QuerySetter` -> `HeaderSetter` -> `BodySetter` -> `Sender` 的链式调用约束请求配置顺序。
- **内部结构**：`sender` 负责维护请求状态（`options`）并执行最终的 `http.Client` 调用；`receiver` 封装 `http.Response`，提供便捷的 Body 解析方法（JSON, XML, Protobuf, Gob 等）。
- **扩展机制**：支持通过 `client.Middleware` 在 `Method` 阶段注入拦截器，实现日志、重试等横切关注点。
- **依赖方向**：依赖 `github.com/soyacen/goose/client` 进行请求调度，依赖 `google.golang.org/protobuf` 和 `github.com/google/go-querystring` 处理特定格式序列化。
