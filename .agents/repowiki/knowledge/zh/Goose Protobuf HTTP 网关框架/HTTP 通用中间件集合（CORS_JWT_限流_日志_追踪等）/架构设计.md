- 扁平子包结构：每个中间件独立一个子包（`cors`, `jwtauth`, `limiter`, `otel`, `errorlog`, `accesslog`, `basicauth`, `context`, `recovery`, `redirect`, `timeout`），内部以 `middleware.go` 暴露入口，复杂模块再拆出 `option.go`（如 `cors`, `errorlog`）或算法文件（如 `limiter/bbr.go`, `limiter/cpu.go`）。
- 统一接口契约：服务端中间件返回 `server.Middleware`（签名 `func(http.ResponseWriter, *http.Request, http.HandlerFunc)`），客户端中间件返回 `client.Middleware`（签名 `func(*http.Client, *http.Request, client.Invoker) (*http.Response, error)`），由 `goose` 核心在路由层串联。
- 依赖隔离：`jwtauth`, `limiter`, `otel` 各自拥有独立 `go.mod`/`go.sum`，将 `golang-jwt/jwt/v5`、`opentelemetry` 等第三方库下沉到子模块，使用者可按需引入而不污染主模块依赖。
- 配置模式：普遍采用函数式选项（`defaultOptions().apply(opts...)` + `Option func(*options)`），并通过 `init()` 做参数校验与默认值回填（见 `limiter/options` 链式 `defaultOptions()->apply()->init()->newLimiter()`）。
- 响应拦截：需要读取状态码/响应体的中间件通过包装 `http.ResponseWriter`（`statusCodeResponseWriter`、`responseWriter`）覆盖 `WriteHeader`/`Write` 来捕获下游结果，而非修改 handler 返回值。
