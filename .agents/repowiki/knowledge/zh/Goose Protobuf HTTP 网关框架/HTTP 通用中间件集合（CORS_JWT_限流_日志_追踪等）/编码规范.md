- 服务端/客户端中间件入口函数统一命名为 `Server(...)` / `Client(...)`，分别返回 `server.Middleware` / `client.Middleware` 类型。
- 配置采用函数式选项模式：定义 `type options struct` + `type Option func(*options)` + `defaultOptions().apply(opts...)` 三件套，新增参数只需追加 Option 函数即可向后兼容。
- 需要读取下游响应状态的中间件通过组合 `http.ResponseWriter` 并重写 `WriteHeader`/`Write` 来拦截状态码与响应体，而不是修改调用方返回值。
- 对请求体进行二次读取的中间件（`errorlog`, `accesslog`）一律先 `io.ReadAll` 再用 `io.NopCloser(bytes.NewReader(...))` 重新赋值回 `request.Body`，保证后续 handler 仍可消费。
- 可选子包（`jwtauth`, `limiter`, `otel`）各自维护独立 `go.mod`/`go.sum`，将重型第三方依赖下沉到子模块，避免主模块被间接引入。
