- 采用分层设计：`decoder.go` 和 `encoder.go` 负责底层数据序列化（Protobuf/JSON/HTTP Body），`middleware.go` 处理请求拦截与链式调用，`option.go` 通过函数选项模式管理配置。
- 依赖方向：核心逻辑依赖 `google.golang.org/protobuf` 进行消息编解码，依赖 `github.com/soyacen/goose` 定义的错误编码器和上下文工具。
- 接口边界：暴露 `DecodeRequest`、`EncodeResponse` 等纯函数供上层框架调用，`Middleware` 类型定义了标准的 HTTP 中间件签名。
