- 采用函数式选项模式（Functional Options Pattern）进行中间件配置，通过定义 `Option` 函数类型和 `defaultOptions().apply()` 方法实现灵活且向后兼容的参数设置。
- 遵循统一的中间件构造器命名规范，服务端中间件入口函数统一命名为 `Server`，客户端中间件入口函数统一命名为 `Client`，返回对应的 `server.Middleware` 或 `client.Middleware` 类型。
- 在需要捕获响应状态码或响应体的中间件（如 `accesslog`, `limiter`）中，通过包装 `http.ResponseWriter` 接口（如 `statusCodeResponseWriter`）来拦截 `WriteHeader` 和 `Write` 调用，从而获取下游处理结果。
