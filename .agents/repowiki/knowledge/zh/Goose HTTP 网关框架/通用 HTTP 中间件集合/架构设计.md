- **模块化子包架构**：每个中间件（如 `cors`, `jwtauth`, `limiter`）作为独立子包实现，内部包含 `middleware.go` 及可选的 `option.go` 配置逻辑，部分复杂模块（如 `limiter`）进一步拆分算法实现（`bbr.go`）。
- **统一接口契约**：服务端中间件遵循 `server.Middleware` 签名 (`func(http.ResponseWriter, *http.Request, http.HandlerFunc)`)，客户端中间件遵循 `client.Middleware` 签名，确保与 `goose` 框架核心无缝集成。
- **配置模式**：广泛采用函数式选项模式（Functional Options），通过 `defaultOptions().apply(opts...)` 链式调用初始化配置，保持 API 简洁且可扩展。
- **依赖隔离**：特定功能模块（如 `jwtauth`, `limiter`, `otel`）拥有独立的 `go.mod`，实现依赖解耦，允许使用者仅引入所需中间件的第三方库（如 `golang-jwt`, `opentelemetry`）。
