- **编译期与运行期解耦**：`protoc_gen_goose` 在编译期将 `.proto` 映射为 Go 代码与 OpenAPI 文档；`server`、`client` 及 `middleware` 在运行期提供基于根包契约的编解码、路由注册及链路处理能力。
- **根包核心契约**：根模块 `goose` 定义了跨子模块共享的基础设施，包括 `RouteInfo`（上下文路由信息注入）、`ErrorEncoder/Decoder`（统一错误序列化）及 `ValidateRequest` 接口，确保生成代码与手动中间件在状态码映射和错误处理上的一致性。
- **标准化数据流转**：所有子模块依赖根包定义的 `http.ResponseWriter` 处理逻辑与 `context.Context` 传递机制，利用 `url.Values` 作为路径、查询及表单数据的内部中转格式，实现从 HTTP 请求到 Protobuf 消息的标准化映射。
