- **入口与编排**：`main.go` 实现 `protogen.Plugin` 接口，解析 `.proto` 文件中的 Service 定义，并协调各子模块生成代码。
- **语义解析层**：`parser/` 包负责提取 gRPC 服务元数据，通过 `annotations.HttpRule` 解析 HTTP 映射规则（Method, Path, Body），并将流式调用标记为不支持。
- **代码生成层**：采用职责分离设计，`server/` 生成基于 `http.ServeMux` 的路由注册与请求解码逻辑；`client/` 生成包含中间件链、编码器和解码器的 HTTP 客户端实现。
- **文档生成层**：`openapi/` 包在启用 `-openapi` 标志时，将解析后的端点信息转换为 OpenAPI 3.0.3 规范的 JSON 文档。
- **常量管理**：`constant/` 集中管理所有外部依赖包的 `protogen.GoImportPath` 和标识符引用，确保生成代码的导入路径一致性。
