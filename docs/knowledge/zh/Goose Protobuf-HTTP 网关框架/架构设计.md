- **代码生成与运行时分离**：`protoc_gen_goose` 负责在编译期根据 `.proto` 文件生成服务端 Handler、客户端 SDK 及 OpenAPI 文档；根目录下的 `server`、`client` 及 `middleware` 模块提供运行时所需的编解码器、路由注册及中间件链。
- **统一的数据契约**：根包 `goose` 定义了跨模块共享的基础设施，包括 `RouteInfo`（上下文路由信息注入）、`ErrorEncoder/Decoder`（统一的 HTTP 错误序列化协议）以及 `ValidateRequest`（基于接口的 Protobuf 消息校验）。
- **标准化交互边界**：所有子模块均依赖根包定义的 `http.ResponseWriter` 处理逻辑与 `context.Context` 传递机制，确保生成的代码与手动编写的中间件在错误处理和状态码映射上保持一致。
